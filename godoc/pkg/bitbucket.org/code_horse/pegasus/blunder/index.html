    type Manager ¶

    Manager describes the error which will handle all the error according to the following specific flags. Some of the flag could be importance, priority, store, fatal, etc... The Error manager will send the error at third party provider if needed in order to store it or analyze it. Those third party provider could be local micro-services.

    func Set ¶ 

    func Set(message string, err error) *Manager

    Set the error message and the error object. The message could pass as string parameter as the error object. The function will return a Manager object. The error manager struct embedded the importance stuck in order to be able to set the error. So after e := error.Manager(...) we could use the following practise e.SetFatal() in order to set the importance.

    func (*Manager) Handle ¶ 

    func (m *Manager) Handle()

    Handle the error according to struct properties. If the err object is nill the the handle function will silence quit, else will check each case of importance.

    func (*Manager) SetError ¶ 

    func (i *Manager) SetError()

    Something is broken could be a http call with other service, could be a validation param, etc. We store this message and stop the execute of the current call.

    func (*Manager) SetFatal ¶ 

    func (i *Manager) SetFatal()

    Something is go wrong and it has to be with very sensitive data. Could be a Payment transaction or wrong configurations for pricing, etc.. We store this error and stop the execution of current call. Also we notify the programmers immediately.

    func (*Manager) SetInfo ¶ 

    func (i *Manager) SetInfo()

    Is the info we only will send the error in order to store it and nothing else will happen.

    func (*Manager) SetPanicError ¶ 

    func (i *Manager) SetPanicError()

    Something happens and the application is unable to start. We stop it and then we throw a panic alert.

    func (*Manager) SetWarring ¶ 

    func (i *Manager) SetWarring()

    Something is wrong and may accrue a problem. We send this error and store it.
